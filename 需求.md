# Obsidian 健身记录插件需求文档

版本日期：2026-06-26

## 1. 项目目标

实现一个 Obsidian 插件，用于在 Vault 中记录、维护和可视化个人健身数据。

插件采用“单一 Markdown 数据文件 + 自定义插件界面”的方式：数据保存在 Vault 内的一个 `.md` 文件中，插件读取并渲染为可交互的健身记录工作台。

首版目标：

- 左侧动作库：管理训练板块和动作标签。
- 中央主展示区：按日期条目式展示训练记录。
- 右侧热力图与体重图：按动作训练次数展示肌肉热力，并同步显示体重变化。
- 实时保存：所有操作写回单一 Markdown 数据文件。
- 实施同步：开发过程中持续更新本文档的完成情况和需求变更。

## 2. 可行性评估

该需求可行。Obsidian 插件可以实现自定义 View、Ribbon 图标、命令、Vault 文件读写、弹窗、样式和交互组件。

推荐技术路线：

- 使用 TypeScript 编写 Obsidian 插件。
- 使用一个 Markdown 文件保存结构化 YAML 数据。
- 插件启动后注册一个专用 View。
- View 中渲染主展示区、右侧热力图侧边栏。
- 数据变更后写回 Markdown 文件。

主要风险：

- 表格列配置和行拖拽复杂度较高。
- 热力图必须先建立稳定的肌肉枚举与映射。
- 移动端对拖拽和密集表格交互不友好，需要降级。
- 单文件数据需要 schema 版本和迁移策略，避免后续无法兼容。

## 3. 功能需求

### 3.1 动作库侧边栏

动作库作为左侧可折叠侧边栏展示。

功能要求：

- 按大肌肉群或功能划分板块，例如胸、肩、背、腿、核心、有氧。
- 板块允许用户新增。
- 每个板块标题可点击折叠或展开。
- 每个板块右上角有新增动作按钮。
- 所有板块底部有“添加板块”按钮。
- 每个动作以标签形式展示，只显示动作名称。
- 点击动作标签弹出编辑框。

新增或编辑动作时包含：

- 动作名称。
- 锻炼肌肉。
- 详细说明。

锻炼肌肉输入要求：

- 支持通过输入匹配预设肌肉。
- 预设肌肉必须与热力图可展示区域一致。
- 示例：输入“肱”时，匹配所有包含“肱”的选项，如肱二头肌、肱三头肌。
- 动作名称为空时不添加。

### 3.2 健身记录展示

中央区域展示按日期分组的训练条目，参考记账类应用的时间线形态。

基础交互：

- 页面顶部提供搜索、动作库入口和新增记录按钮。
- 搜索支持中文输入法组合输入。
- 每个动作单独显示为一个条目。
- 日期标题后显示当天最新体重。
- 条目支持编辑和删除。

记录弹窗：

- 日期通过日期选择器选择，默认不允许手输。
- 体重默认取上一次填写的体重。
- 动作按肌肉板块标签页切换。
- 每个动作可填写组数、每组个数、重量。

### 3.3 热力展示

热力展示作为右侧可折叠侧边栏。

功能要求：

- 展示人体正面和背面肌肉结构。
- 首版使用插件内置资源或内置 CSS 区域图形实现。
- 统计记录表中每个健身动作出现的次数。
- 根据动作对应肌肉累计次数。
- 次数越多，对应肌肉区域颜色越深。
- 肌肉预设值必须与动作编辑中的肌肉选项一致。
- 时间范围支持 30 / 90 / 180 / 365 / 全部。
- 热力图下方展示体重变化折线图，时间范围与热力图一致。
- 右侧侧边栏支持固定、收起、点击窄条展开和手动拉伸宽度。

### 3.4 数据文件

使用一个 Markdown 文件记录数据并由插件渲染。

默认文件：

```text
fitness-record.md
```

数据文件内容以 YAML 结构化数据为主，包含：

- schema 版本。
- 板块列表。
- 动作列表。
- 表格列配置。
- 训练记录。
- UI 折叠状态和热力图时间范围。

### 3.5 交互和性能

要求：

- 每个弹窗右上角都有关闭按钮。
- 所有按钮都必须有响应。
- 加载速度快，不占用过多系统资源。
- 数据变更后及时保存。
- 保存失败时显示 Obsidian Notice。
- 可参考 `D:\myFiles\Project\apex-dashboard` 的插件结构、样式组织和单文件数据思路。

## 4. 代码审查与潜在问题处理

### 4.1 数据结构风险

风险：单文件数据结构如果没有版本，会导致后期迁移困难。

处理：

- 数据 schema 必须包含 `schemaVersion`。
- 读取旧数据时走 normalize/migrate 流程。
- 写回时统一输出当前 schema。

### 4.2 拖拽交互风险

风险：行拖拽、点击行菜单、单元格编辑可能互相抢事件。

处理：

- 只允许通过行把手拖拽。
- 单元格点击只负责编辑。
- 行删除通过明确按钮触发。
- 移动端首版可降级为上移/下移或暂不支持拖拽。

### 4.3 热力图映射风险

风险：动作填写的肌肉名称与热力图区域不一致，会导致统计无法展示。

处理：

- 维护一份统一肌肉枚举。
- 动作只能选择枚举内肌肉。
- 热力图只按枚举 id 统计。

### 4.4 性能风险

风险：每次输入都全量重绘和写文件，数据量大时会卡。

处理：

- 首版限制在用户操作完成后保存。
- 渲染采用局部重建主区域，避免频繁监听键入写入。
- 后续可增加防抖保存。

### 4.5 移动端风险

风险：移动端屏幕窄，三栏和复杂表格难用。

处理：

- 窄屏改为上下堆叠布局。
- 左右侧边栏折叠按钮始终可见。
- 表格区域允许横向滚动。

### 4.6 Obsidian 主题兼容风险

风险：不同主题颜色变量不同，导致按钮或表格不可读。

处理：

- 优先使用 Obsidian CSS 变量。
- 自定义颜色保持足够对比度。
- 深色/浅色都要检查。

## 5. 验收标准

- 插件能在 Obsidian 中启用。
- 可通过 Ribbon 图标或命令打开健身记录视图。
- 首次打开时自动创建默认数据文件。
- 能通过动作库弹窗新增、编辑动作和肌肉板块。
- 能通过添加记录弹窗新增训练记录。
- 能按日期条目式展示记录，并编辑或删除记录。
- 热力图能根据记录动作统计更新。
- 体重变化折线图能跟随热力图时间范围更新。
- 右侧热力图侧边栏能固定、收起、展开和手动拉伸。
- 重启 Obsidian 后数据仍能恢复。
- 所有弹窗有关闭按钮。
- 主要按钮均能响应。
